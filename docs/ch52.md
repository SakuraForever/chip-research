## 附录A：RTL 实现示例（Chisel / Verilog）

>  **本章目标**：提供 FlexNPU 关键模块的可综合 RTL 代码示例，让芯片从业者能够直接理解从架构到实现的映射。

### A.1 INT8 MAC 单元（Chisel + Verilog 对照）

**Chisel 实现**：

```scala
import chisel3._
import chisel3.util._

/** INT8 乘累加单元
  * 输入: a(8-bit), b(8-bit), acc_in(32-bit)
  * 输出: acc_out(32-bit) = acc_in + a * b
  * 延迟: 1 cycle (组合逻辑乘法 + 寄存器累加)
  */
class Int8MAC extends Module {
  val io = IO(new Bundle {
    val a      = Input(SInt(8.W))    // 激活值 (signed INT8)
    val b      = Input(SInt(8.W))    // 权重 (signed INT8)
    val acc_in = Input(SInt(32.W))   // 累加输入
    val acc_out= Output(SInt(32.W))  // 累加输出
    val valid  = Input(Bool())       // 使能信号
  })

  // 乘法: 8×8 → 16-bit (组合逻辑)
  val product = io.a * io.b  // SInt(16.W)

  // 累加: 16 + 32 → 32-bit (寄存器)
  val acc_reg = RegInit(0.S(32.W))

  when(io.valid) {
    acc_reg := io.acc_in + product
  }

  io.acc_out := acc_reg
}
```

**Verilog 等效实现**：

```verilog
module Int8MAC (
    input  wire         clk,
    input  wire         rst_n,
    input  wire signed [7:0]   a,       // 激活值
    input  wire signed [7:0]   b,       // 权重
    input  wire signed [31:0]  acc_in,  // 累加输入
    output reg  signed [31:0]  acc_out, // 累加输出
    input  wire         valid          // 使能信号
);

wire signed [15:0] product;

// 乘法 (组合逻辑)
assign product = a * b;

// 累加 (时序逻辑)
always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        acc_out <= 32'sd0;
    else if (valid)
        acc_out <= acc_in + product;
end

endmodule
```

> **面积参考** (7nm): ~400 μm² | **延迟**: ~0.3ns (INT8) | **功耗**: ~0.02mW/MHz

### A.2 PE 5级流水线（Chisel 实现）

```scala
/** FlexNPU PE (Processing Element)
  * 5级流水线: IF → ID → MAC → ACC → WB
  * 支持: INT8 / FP16 双精度
  * 数据流: Weight Stationary (WS)
  */
class FlexNPU_PE extends Module {
  val io = IO(new Bundle {
    // 数据输入 (来自SRAM或邻居PE)
    val act_in     = Input(SInt(8.W))    // 激活值输入
    val weight_in  = Input(SInt(8.W))    // 权重输入
    val psum_in    = Input(SInt(32.W))   // 部分和输入 (来自上方PE)

    // 数据输出 (传递给邻居PE)
    val act_out    = Output(SInt(8.W))   // 激活值传递 (水平)
    val weight_out = Output(SInt(8.W))   // 权重传递 (垂直)
    val psum_out   = Output(SInt(32.W))  // 部分和传递 (下方)

    // 控制
    val opcode     = Input(UInt(3.W))    // 操作码
    val valid_in   = Input(Bool())       // 输入有效
    val valid_out  = Output(Bool())      // 输出有效
    val mode       = Input(UInt(2.W))    // 0=INT8, 1=FP16, 2=空闲
  })

  // ===== Stage 1: Input Register (IF) =====
  val act_reg   = RegInit(0.S(8.W))
  val wgt_reg   = RegInit(0.S(8.W))
  val psum_reg  = RegInit(0.S(32.W))
  val valid_reg = RegInit(false.B)

  when(io.valid_in) {
    act_reg  := io.act_in
    wgt_reg  := io.weight_in
    psum_reg := io.psum_in
  }
  valid_reg := io.valid_in

  // ===== Stage 2: Decode (ID) =====
  val do_mac = (io.opcode === 1.U) & valid_reg  // MAC指令
  val do_nop = (io.opcode === 0.U)               // NOP

  // ===== Stage 3: Multiply (MAC) =====
  val product = act_reg * wgt_reg  // 8×8 → 16-bit

  // ===== Stage 4: Accumulate (ACC) =====
  val result = RegInit(0.S(32.W))
  when(do_mac) {
    result := psum_reg + product
  }

  // ===== Stage 5: Writeback (WB) + 脉动传递 =====
  io.act_out    := act_reg       // 激活值水平传递
  io.weight_out := wgt_reg       // 权重垂直传递
  io.psum_out   := Mux(do_mac, result, psum_reg)  // 部分和下方传递
  io.valid_out  := valid_reg
}
```

**PE 关键时序**：

| 阶段 | 功能 | 延迟(cycle) | 关键路径 |
|------|------|-----------|---------|
| IF | 输入寄存 | 1 | — |
| ID | 操作码解码 | 1 | — |
| MAC | 8×8 乘法 | 1 | 乘法器组合逻辑 ~0.2ns |
| ACC | 32-bit 累加 | 1 | 加法器 + MUX ~0.15ns |
| WB | 输出寄存 | 1 | — |

### A.3 16×16 Weight Stationary 脉动阵列

```scala
/** 16×16 Weight Stationary 脉动阵列
  * 权重预加载后保持不变，激活值从左侧流入，部分和从上方流入
  * 结果从下方流出
  */
class SystolicArray16x16 extends Module {
  val io = IO(new Bundle {
    // 激活值输入 (16列, 从左侧)
    val act_in   = Input(Vec(16, SInt(8.W)))
    // 部分和输入 (16列, 从上方)
    val psum_in  = Input(Vec(16, SInt(32.W)))
    // 权重加载 (16×16, 逐行加载)
    val weight_ld= Input(Vec(16, SInt(8.W)))
    val weight_en= Input(Bool())      // 权重加载使能
    val row_sel  = Input(UInt(4.W))   // 行选择
    // 结果输出 (16列, 从下方)
    val psum_out = Output(Vec(16, SInt(32.W)))
    // 控制
    val valid_in = Input(Bool())
    val valid_out= Output(Vec(16, Bool()))
  })

  // 16×16 PE 阵列
  val pes = Seq.fill(16, 16)(Module(new FlexNPU_PE))

  // 权重寄存器 (Weight Stationary: 预加载)
  val weights = RegInit(VecInit(Seq.fill(16, 16)(0.S(8.W))))

  // 权重加载
  when(io.weight_en) {
    for (c <- 0 until 16) {
      weights(io.row_sel)(c) := io.weight_ld(c)
    }
  }

  // 连接 PE 阵列
  for (r <- 0 until 16; c <- 0 until 16) {
    val pe = pes(r)(c)

    // 权重: 固定 (Weight Stationary)
    pe.io.weight_in := weights(r)(c)

    // 激活值: 水平传递 (左 → 右)
    if (c == 0) {
      pe.io.act_in := io.act_in(r)
    } else {
      pe.io.act_in := pes(r)(c - 1).io.act_out
    }

    // 部分和: 垂直传递 (上 → 下)
    if (r == 0) {
      pe.io.psum_in := io.psum_in(c)
    } else {
      pe.io.psum_in := pes(r - 1)(c).io.psum_out
    }

    // 控制
    pe.io.opcode   := Mux(io.valid_in, 1.U, 0.U)  // 1=MAC, 0=NOP
    pe.io.valid_in := io.valid_in
    pe.io.mode     := 0.U  // INT8

    // 输出: 最底行
    if (r == 15) {
      io.psum_out(c)  := pe.io.psum_out
      io.valid_out(c) := pe.io.valid_out
    }
  }
}
```

**数据流示意**：

```
Cycle:    0    1    2    3   ...  15   16   ...  31
         ┌───┐
act[0] → │PE │ → act[0]' → PE → ... 
psum↓    │0,0│              ↓
         └───┘            psum'
act[1] →  PE  → ...      PE
          ↓                ↓
         ...              ...
act[15]→ PE  → ...      PE → psum_out[15]
                                       
权重 w[r][c] 在计算前预加载，计算期间保持不变
激活 a[r] 从左侧逐周期流入，水平传递
部分和从上方流入，垂直累加后从下方流出
```

### A.4 FlashAttention FSA Tile 核心逻辑

```scala
/** FSA (FlashAttention Systolic Array) Tile
  * 参考: SystolicAttention (EPFL), arxiv 2025
  * 在标准脉动阵列基础上增加:
  *   - Split 单元: 近似 Softmax 的指数计算
  *   - CMP row: 行最大值追踪 (用于数值稳定)
  *   - UpPath: Attention Value 反向路径
  */
class FSATile16x16 extends Module {
  val io = IO(new Bundle {
    // Q, K, V 输入
    val q_in  = Input(Vec(16, SInt(8.W)))   // Query (行广播)
    val k_in  = Input(Vec(16, SInt(8.W)))   // Key (列加载)
    val v_in  = Input(Vec(16, SInt(8.W)))   // Value (列加载)

    // 控制信号
    val phase = Input(UInt(2.W))  // 0=Score(QK^T), 1=Softmax, 2=Value
    val valid  = Input(Bool())

    // 输出
    val attn_out = Output(Vec(16, SInt(16.W)))  // Attention 输出
    val out_valid= Output(Bool())
  })

  // 基础 16×16 脉动阵列
  val systolic = Module(new SystolicArray16x16)

  // ===== Phase 0: Attention Score (QK^T) =====
  // 复用标准脉动阵列，Q 行广播，K 列加载
  // 输出: S[i][j] = Q[i] · K[j]

  // ===== Split 单元: Softmax 近似 =====
  // Softmax(x) = exp(x - max(x)) / Σ exp(x - max(x))
  // Split 近似: exp(x) ≈ 2^(x) 的分数部分线性插值
  val max_val  = RegInit(VecInit(Seq.fill(16)(0.S(16.W))))
  val sum_exp  = RegInit(VecInit(Seq.fill(16)(0.S(16.W))))
  val exp_lut  = Mem(256, SInt(16.W))  // exp 查找表 (预计算)

  // ===== CMP row: 行最大值追踪 =====
  // 每个 cycle 比较16个PE的输出，保留最大值
  for (c <- 0 until 16) {
    val current = systolic.io.psum_out(c)
    when(current > max_val(c)) {
      max_val(c) := current
    }
  }

  // ===== Phase 2: Attention × Value =====
  // Softmax(S) × V，复用脉动阵列
  // UpPath: 将 Softmax 结果反馈回阵列乘以 V

  // 输出 MUX
  io.attn_out := VecInit.tabulate(16)(i => systolic.io.psum_out(i)(15,0).asSInt)
  io.out_valid := systolic.io.valid_out(0)
}
```

**FSA vs 标准脉动阵列面积开销**：

| 组件 | 标准PE面积 | FSA新增面积 | 开销 |
|------|----------|-----------|------|
| MAC + 寄存器 | 1600 μm² | — | 基准 |
| Split 单元 | — | ~120 μm² | +7.5% |
| CMP row | — | ~80 μm² | +5% |
| UpPath | — | ~40 μm² | +2.5% |
| Exp LUT (256×16) | — | ~60 μm² | +3.75% |
| **总计** | **1600 μm²** | **~300 μm²** | **+18.75%** |

> **参考**: FSA论文报告 +12% 面积开销（128×128阵列，分摊后更小）

### A.5 DMA 引擎接口

```scala
/** FlexNPU DMA 引擎
  * 支持: 2D 传输 (带stride)、多通道仲裁
  * 接口: AXI4 Master (连接 NoC)
  */
class FlexNPU_DMA extends Module {
  val io = IO(new Bundle {
    // AXI4 Master 接口
    val axi = new AXI4MasterIO(addrWidth = 64, dataWidth = 128)

    // DMA 描述符接口 (CSR写入)
    val desc_src_addr  = Input(UInt(64.W))   // 源地址 (DDR)
    val desc_dst_addr  = Input(UInt(64.W))   // 目的地址 (SRAM)
    val desc_x_size    = Input(UInt(16.W))   // 每行字节数
    val desc_y_size    = Input(UInt(16.W))   // 行数
    val desc_src_stride= Input(UInt(16.W))   // 源行间距
    val desc_dst_stride= Input(UInt(16.W))   // 目的行间距
    val desc_start     = Input(Bool())       // 启动传输
    val desc_done      = Output(Bool())      // 传输完成
  })

  // DMA 状态机
  val sIDLE :: sREAD :: sWRITE :: sDONE :: Nil = Enum(4)
  val state = RegInit(sIDLE)

  val x_cnt = RegInit(0.U(16.W))
  val y_cnt = RegInit(0.U(16.W))
  val src_ptr = RegInit(0.U(64.W))
  val dst_ptr = RegInit(0.U(64.W))

  switch(state) {
    is(sIDLE) {
      when(io.desc_start) {
        state   := sREAD
        src_ptr := io.desc_src_addr
        dst_ptr := io.desc_dst_addr
        x_cnt   := 0.U
        y_cnt   := 0.U
      }
    }
    is(sREAD) {
      // 发起 AXI4 读请求
      io.axi.ar.valid := true.B
      io.axi.ar.addr  := src_ptr
      when(io.axi.ar.ready) {
        src_ptr := src_ptr + 16.U  // 128-bit = 16 bytes
        x_cnt   := x_cnt + 1.U
        when(x_cnt === (io.desc_x_size >> 4.U)) {
          // 一行读完，切换到下一行
          x_cnt   := 0.U
          y_cnt   := y_cnt + 1.U
          src_ptr := src_ptr - x_cnt + io.desc_src_stride
          state   := Mux(y_cnt === io.desc_y_size, sDONE, sREAD)
        }
      }
    }
    is(sDONE) {
      state := sIDLE
    }
  }

  io.desc_done := (state === sDONE)
}
```

### A.6 FlexNPU Tile 配置寄存器文件

```scala
/** FlexNPU CSR (Control and Status Register) 文件
  * 地址映射: 0x4000_1000 ~ 0x4000_1FFF
  */
class FlexNPU_CSR extends Module {
  val io = IO(new Bundle {
    // CPU 访问接口 (AXI4-Lite Slave)
    val csr_addr  = Input(UInt(12.W))
    val csr_wdata = Input(UInt(32.W))
    val csr_wen   = Input(Bool())
    val csr_rdata = Output(UInt(32.W))

    // 到 NPU 内部的控制信号
    val npu_mode     = Output(UInt(2.W))     // 0=CNN, 1=Attention, 2=Hybrid
    val npu_tile_sz  = Output(UInt(3.W))     // 0=16, 1=32, 2=64
    val npu_prec     = Output(UInt(2.W))     // 0=INT4, 1=INT8, 2=FP16, 3=FP32
    val npu_start    = Output(Bool())
    val npu_done     = Input(Bool())
    val npu_status   = Input(UInt(3.W))      // 0=空闲, 1=运行, 2=完成, 3=错误

    // 性能计数器输出
    val perf_mac_util  = Input(UInt(32.W))
    val perf_bw_util   = Input(UInt(32.W))
    val perf_cycle     = Input(UInt(32.W))
  })

  // CSR 地址映射
  val REG_MODE      = 0x000   // R/W: 运行模式
  val REG_TILE_SIZE = 0x004   // R/W: Tile 大小
  val REG_PRECISION = 0x008   // R/W: 精度
  val REG_START     = 0x00C   // WO: 启动 (写1触发)
  val REG_STATUS    = 0x010   // RO: 状态
  val REG_PERF_MAC  = 0x014   // RO: MAC利用率
  val REG_PERF_BW   = 0x018   // RO: 带宽利用率
  val REG_PERF_CYC  = 0x01C   // RO: 总周期数

  // 寄存器
  val reg_mode     = RegInit(0.U(2.W))
  val reg_tile     = RegInit(0.U(3.W))
  val reg_prec     = RegInit(1.U(2.W))  // 默认INT8

  // 写逻辑
  when(io.csr_wen) {
    switch(io.csr_addr) {
      is(REG_MODE.U)      { reg_mode := io.csr_wdata(1, 0) }
      is(REG_TILE_SIZE.U) { reg_tile := io.csr_wdata(2, 0) }
      is(REG_PRECISION.U) { reg_prec := io.csr_wdata(1, 0) }
    }
  }

  // 读逻辑
  io.csr_rdata := MuxLookup(io.csr_addr, 0.U)(Seq(
    REG_MODE.U      -> reg_mode,
    REG_TILE_SIZE.U -> reg_tile,
    REG_PRECISION.U -> reg_prec,
    REG_STATUS.U    -> io.npu_status,
    REG_PERF_MAC.U  -> io.perf_mac_util,
    REG_PERF_BW.U   -> io.perf_bw_util,
    REG_PERF_CYC.U  -> io.perf_cycle,
  ))

  // 输出
  io.npu_mode    := reg_mode
  io.npu_tile_sz := reg_tile
  io.npu_prec    := reg_prec
  io.npu_start   := io.csr_wen && (io.csr_addr === REG_START.U)
}
```

### A.7 面积与时序综合参考

**基于 Chisel 生成的 Verilog 综合结果 (16nm, Synopsys DC)**：

| 模块 | 面积 (μm²) | 时序 (ns) | 功耗 (mW/MHz) | 关键路径 |
|------|-----------|----------|--------------|---------|
| Int8MAC | ~400 | 0.30 | 0.02 | 乘法器 |
| FlexNPU_PE | ~1,600 | 0.50 | 0.05 | MAC+MUX |

> **注**: 以上数据为综合估算值，基于Gemmini开源项目的公开面积数据推算，并按16nm工艺缩放。实际面积取决于具体工艺库和综合约束。
