```mermaid

    participant RTL as RTL仿真<br/>(Verilator/VCS)

    Note over PY,HW: 逐层对比, 确保一致性

    PY->>COMP: 模型 + 量化方案
    COMP->>COMP: 编译生成指令流
    COMP->>CMODEL: 指令 + 量化权重
    
    CMODEL->>CMODEL: 位精确模拟<br/>(INT8/FP16)
    CMODEL-->>PY: 对比中间结果<br/>(误差 < 1e-3)
    
    COMP->>RTL: 指令流 + 权重
    RTL->>RTL: 仿真执行<br/>(波形+Counter)
    RTL-->>CMODEL: 对比每周期结果<br/>(必须完全一致)
    
    COMP->>HW: 指令二进制
    HW->>HW: 实际执行
    HW-->>RTL: 对比输出<br/>(硬件vs仿真)
```

### 23.4 AXI 总线接口详解

```mermaid
sequenceDiagram
    participant CPU as CPU<br/>(AXI Master)
    participant NPU as NPU<br/>(AXI Slave for CSR)
    participant DMA as DMA<br/>(AXI Master for DDR)
    participant DDR as LPDDR5

    Note over CPU,NPU: 1. 任务下发 (AXI-Lite 写)
    CPU->>NPU: AW: 写地址 0x4000_1000<br/>(CMD_QUEUE)
    CPU->>NPU: W: 写数据 [指令字]<br/>(启动计算)
    NPU-->>CPU: B: 写响应 OKAY

    Note over NPU,DDR: 2. NPU 执行 (DMA 通过 AXI 访问 DDR)
    NPU->>DMA: 内部: 启动DMA传输
    DMA->>DDR: AR: 读请求 (权重/激活)
    DDR-->>DMA: R: 读数据返回
    DMA->>NPU: 内部: 数据写入SRAM
    
    Note over NPU,DDR: 3. 计算完成后写回
    NPU->>DMA: 内部: 启动DMA写回
    DMA->>DDR: AW: 写请求 (输出结果)
    DDR-->>DMA: B: 写响应

    Note over NPU,CPU: 4. 中断通知
    NPU->>CPU: IRQ: 任务完成中断<br/>(或CPU轮询STATUS寄存器)
    CPU->>NPU: AR: 读 0x4000_1004<br/>(STATUS寄存器)
    NPU-->>CPU: R: 状态 = DONE
```

**NPU AXI 接口配置**：

| 接口 | 协议 | 位宽 | 频率 | 用途 |
|------|------|------|------|------|
| **CSR Slave** | AXI4-Lite | 32-bit | 500 MHz | CPU 配置 NPU |
| **DMA Master** | AXI4 | 128-bit | 500 MHz | NPU 访问 DDR |
| **NoC Master** | AXI4 | 128-bit | 1 GHz | NPU 通过 NoC |

### 23.6 硅后 Bring-up 流程

```mermaid
stateDiagram-v2
    [*] --> PowerOn: 上电
    PowerOn --> JTAG: JTAG 连接<br/>(边界扫描)
    JTAG --> BIST: MBIST/SCAN<br/>(自测试)
    BIST --> Boot: 通过 → 安全启动<br/>失败 → 诊断
    Boot --> Driver: Linux 驱动加载<br/>(NPU 设备驱动)
    Driver --> Simple: 简单算子测试<br/>(GEMM/Conv)
    Simple --> Model: 完整模型测试<br/>(BEVFormer)
    Model --> Perf: 性能调优<br/>(Counter + Roofline)
    Perf --> Release: 达标 → 释放<br/>未达标 → RTL修复
    Release --> [*]
    
    诊断 --> [*]: 良率分析
```

**Bring-up 工具链**：

| 阶段 | 工具 | 目的 |
|------|------|------|
| **JTAG** | OpenOCD + GDB | 边界扫描、寄存器读写 |
| **BIST** | 自研脚本 | SRAM 缺陷诊断 |
| **Driver** | Linux kernel module | `/dev/flexnpu` 设备驱动 |
| **Simple Test** | C/C++ 测试程序 | 单算子精度验证 |
| **Model Test** | FlexCompiler + Runtime | 完整模型端到端验证 |
| **Perf Tuning** | FlexProfiler | Counter采集 + Roofline分析 |

<div class="callout callout-insight">

**硅后调试的"黄金法则"**：80% 的硅后问题与**时序**有关（setup/hold violation），而非功能错误。因此 FPGA 原型验证极其重要——它能在流片前发现大部分时序相关 bug。
